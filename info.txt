# 🔥 1. Class Loader Subsystem

### 🔑 Role:

Loads `.class` files into memory

---

## 📦 Types of Class Loaders

| Loader      | Work                                    |
| ----------- | --------------------------------------- |
| Bootstrap   | Loads core Java classes (`java.lang.*`) |
| Extension   | Loads external libraries                |
| Application | Loads your code                         |

---

## ⚙️ Steps:

1. **Loading** → reads `.class`
2. **Linking**

   * Verify (security)
   * Prepare (allocate memory)
   * Resolve (references)
3. **Initialization** → static variables initialized

---

# 🔥 2. Runtime Data Areas (Memory)

This is the **heart of JVM**

---

## 🧱 (A) Heap (Shared)

👉 Stores **objects**

```java
Calculator c = new Calculator();
```

✔ Object goes into **Heap**

---

### 📌 Heap is divided into:

* Young Generation

  * Eden
  * Survivor spaces
* Old Generation (long-living objects)

👉 Used heavily by **Garbage Collector**

---

## 🧱 (B) Stack (Per Thread)

👉 Stores:

* Method calls
* Local variables

```java
int x = 10;
```

✔ Stored in **stack**

---

### 💥 Important:

* Each thread has its own stack
* Stack overflow → infinite recursion

---

## 🧱 (C) Method Area (Shared)

👉 Stores:

* Class metadata
* Static variables
* Method code

---

## 🧱 (D) PC Register

👉 Tracks current instruction of thread

---

## 🧱 (E) Native Method Stack

👉 For native code (C/C++ via JNI)

---

# 🔥 3. Execution Engine

### 🔑 Role:

Executes bytecode

---

## ⚙️ Components:

### 1. Interpreter

* Executes line-by-line
* Slow

---

### 2. JIT Compiler (Just-In-Time)

👉 Converts bytecode → machine code

✔ Speeds up execution
✔ Optimizes frequently used code

---

### 💡 Key Idea:

* First run → interpreted
* Frequent code → compiled by JIT

---

# 🔥 4. Garbage Collector (GC)

### 🔑 Role:

Removes unused objects from heap

---

## 🧠 Example:

```java
Calculator c = new Calculator();
c = null;
```

👉 Object becomes **eligible for GC**

---

## ⚙️ How GC works:

* Finds unused objects
* Frees memory automatically

---

## 📌 Types (High level):

* Minor GC → Young generation
* Major GC → Old generation

---

# 🔥 MEMORY FLOW (IMPORTANT)

```java
Calculator c = new Calculator();
```

### Step-by-step:

1. Class loaded → Method Area
2. `c` → Stack
3. Object → Heap
4. Reference stored in stack pointing to heap

---

# 🎯 FULL EXECUTION FLOW

```text
.java → (JDK: javac) → .class → (JVM)
   → Class Loader
   → Memory Allocation
   → Execution Engine
   → Output
```

---

# ⚡ INTERVIEW LEVEL INSIGHTS

### 🔥 Q: Why Java is fast despite interpretation?

👉 Because of **JIT Compiler**

---

### 🔥 Q: Why no memory leaks like C++?

👉 Because of **Garbage Collector**

---

### 🔥 Q: Stack vs Heap?

| Stack           | Heap    |
| --------------- | ------- |
| Local variables | Objects |
| Fast            | Slower  |
| Per thread      | Shared  |

---

# 🧠 SUPER IMPORTANT CONCEPT

👉 **Reference vs Object**

```java
Calculator c = new Calculator();
```

* `c` → Stack (reference)
* Object → Heap
